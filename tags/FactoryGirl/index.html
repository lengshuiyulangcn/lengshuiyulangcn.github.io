Set Up Test Code for Existing Rails Code Base

3月 20, 2016

工作以后发现一个有趣的事情，很多有经验的程序员擅长在已有的code base上各种修改，各种增加新功能。
但是一旦之前什么基础都没有，那就两眼一抹黑，举步艰难了。我也是其中一个。

很早以前做过很多Rails的项目，这些Rails的项目都不曾有test code。工作以后，在现有的test code基础上渐渐开始依葫芦画瓢写起了test，搞起了所谓的TDD，但是一旦切换到没有test code的项目，瞬间就捉襟见肘，找不着门路了。
索性利用三连休的空隙，彻底上手一下给没有test code的项目添加test code以及测试框架，以弥补以往的不足。

Set Up

前提是已经有了Rails的项目。还未添加任何test代码。
我们选择用Rspec作为test框架。使用FactoryGirl来有效创建dummy的数据库内容。

首先依赖的Gem不要装错了，因为Rspec和FactoryGirl不光能用在Rails框架中，其他的诸如sinatra等基于Rack的web框架基本都能使用，所以，在Rails中我们需要依赖的是以上两个gem。

接下来就是bundle install 以及准备好测试用的数据库了。

factoryGirl的用法

用factoryGirl的DSL很快就能上手如何生成数据库的dummy data。

使用FactoryGirl生成的dummy data也非常方便。

  • 当我们只需要生成和params一样用于传递给post方法的参数的时候我们使用attributes_for方法。

  • 当我们需要生成具体的数据库数据并写入数据库的时候，使用create方法。

  • 党我们只需要生成数据并不需要写入数据库的时候，使用build方法。
