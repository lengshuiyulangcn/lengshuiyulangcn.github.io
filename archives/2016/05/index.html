rack 中间件 warden 的机制

5月 31, 2016

说到warden，大概很多人感觉很陌生。然而很多人耳熟能详的Devise就恰恰是基于warden构建的。
所以当我们customize Devise的一些功能的时候常常看到有warden这个关键词的出现。
要深入了解Devise的验证机制，不如先从了解warden这个东西入手。

概述

warden 通过strategy来定义用户验证策略。

在验证的时候我们可以用到的一些方法有

  • request – Rack::Request object
  • session – The session object for the request
  • params – The parameters of the request
  • env – The Rack env object

比如上面我们用params获取了传来的用户名和密码参数。

进行验证之后，根据验证结果，我们可以有一下处理。

  • halt! – 停止接下来所有的strategy。当前的strategy会作为最后一个策略。
  • pass – 直接忽略次策略。当然不推荐就是了。ignore this strategy. It is not required to call this and this is only sugar.
  • success! – 策略成功。因为warden的策略有一个成功即会停止，所以紧接着会造成halt!也发动。
  • fail! – 策略失败。并发动halt！
  • redirect! – 跳转到另一个url。同样发动halt!
  • custom! – 返回一个自定义的rack array并造成halt!

感悟

当Devise太重的时候我们完全可以通过定制自己的warden strategy来定制我们自己的用户验证机制。

如果猫从世界上消失

5月 14, 2016

左手的知觉已经大部分恢复了。郁郁寡欢的心情也像最近的天气一样逐渐转晴。
今天去看了电影版的「世界から猫が消えたなら」(如果猫从世界上消失)的首映。

就在一切都很平淡的一天，恶魔来到了被突然告知身患脑瘤，生命仅剩一天的男主身边。
作为生命延长一天的代价，世界上会有一样东西消失。
第一天恶魔从世界上带走了电话。男主和前女友因电话相识的回忆在此消失。
第二天恶魔从世界上带走了电影。男主和亲友因为电影而相识，每天交流电影的日常也像从未存在一般。
第三天恶魔决定带走陪伴在男主身边的猫。猫和逝去的母亲的回忆涌上心头。
最终,对人与人之间感情的珍重战胜了死亡的恐惧，男主得以释然微笑面对死亡。

电话，电影，猫，这三样东西无非就是喻指爱情，友情和亲情这三样东西。人和人之间的关系，因为这些东西交织在一起，在生活中无形地留下痕迹。
原著作者借着这么一个文艺的标题，无非就是想要传达，如果这个世界上有比死亡更恐惧的东西，那必定是失去爱情，友情和亲情了吧。

今年的春天很短，有相当一段时间在病院里躺过去了。马上该是黄梅时节了。
每到这个时节伴随的心情低落也是在所难免。甚至有好几次梦中醒来,泪流满面。
那年那时的在雨中无助奔跑的我，而今该有了自信去守护自己重要的东西了吧。
尽管如此，岁月的无情和残酷总是让人唏嘘不已。
